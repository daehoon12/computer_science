# DAO Pattern  

## 문제  
- DB를 사용하는 방법이 변경되면 클라이언트의 수정이 많아짐  

## 해결방안  
- 사용 방법의 분리 (Create, Read, Update, Delete)를 분리해서 구현   

## 결과  
- loose coupling (클래스 의존성 ↓), 확장성
